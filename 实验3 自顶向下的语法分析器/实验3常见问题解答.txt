
问：编译器会提示我pasper.cpp里的一些变量没有定义，但已经在clang.lex里面定义了，是为什么呢？
答：如果你用到了lex.yy.c中定义的全局变量或函数，那么要在parser.cpp中声明。
比如变量的声明：extern int yylval; /*假设yylval变量的类型为int */
比如函数的声明：extern int yylex(void);
因为yylex函数在.c文件中，编译器可能按c方式进行编译，而parser.cpp调用时用C++方式，所以此时要写为
extern "C" int yylex(void);

问：如何让lex.yy.c和parser.cpp能同时访问宏ID_TKN呢？
答：在我建立的lex.yy.h文件中添加宏定义。因为这个文件被lex.yy.c和parser.cpp进行#include，所以这个宏就能被两个文件访问。而且在lex.yy.h文件中修改了宏定义的值，lex.yy.c和parser.cpp就自动改变宏值。

问：测试文件是要放在同一个工程下还是同一个文件夹下呢？
答：你的代码可以使用绝对路径来打开测试文件，此时测试文件可以放在硬盘的任何地方。如果你只想输入测试文件名而不提供路径，那么测试文件就放在工程目录下，因为程序运行时默认当前目录就是工程目录。

问：Dev C++有时不能设断点进行调试，怎么办？
答：让Dev开发工具能够调试的步骤：
    菜单“工具”-->“编译选项”-->"代码生成/优化"-->"连接器"-->“产生调试信息”-->no变成yes
有些时候代码只有放在C盘中操作系统创建的“文档”目录中，Dev才能调试该代码。这是Dev程序的不完善之处。
